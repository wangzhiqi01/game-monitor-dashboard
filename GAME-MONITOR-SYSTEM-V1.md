# 游戏监控系统 V1 方案（网页版仪表盘）

## 1. 项目目标
搭建一套面向游戏行业从业者的网页版监控系统，用于持续监控端游与手游的：

- 版本动态
- 营销动作
- 广告投放
- 热度数据
- 用户口碑
- 竞品对比
- 重点结论与建议

该系统不是单纯“数据大屏”，而是一个可持续维护的**竞品情报 + 产品档案 + 营销监控**平台。

---

## 2. 产品定位

### 一句话定位
一个面向游戏市场/品牌/发行/产品团队的**竞品与版本营销情报仪表盘**。

### 适用场景
- 周会/晨会快速扫盘
- 重点竞品长期跟踪
- 版本节点观察
- 营销案例复盘
- 老板汇报
- 方案前置调研

---

## 3. V1 页面结构

## 3.1 首页总览 Dashboard
用于回答：**这周行业里发生了什么？**

模块建议：
- 今日/本周关键动态数
- 重点监控游戏总数
- 端游/手游占比
- 本周重点版本更新
- 本周重点营销动作
- 本周重点联动/福利/活动
- 风险提醒
- 机会点提醒

---

## 3.2 动态监控页 Activity Feed
用于回答：**每个游戏最近做了什么动作？**

字段建议：
- 日期
- 游戏名称
- 平台（端游/手游）
- 状态（已上线/预约/测试）
- 动态类型
  - 版本更新
  - 活动福利
  - 联动合作
  - 直播/视频传播
  - 广告投放
  - 口碑舆情
  - 排名/热度变化
- 动态摘要
- 来源链接
- 重要级别
- 我方启发/备注

筛选能力：
- 按平台筛选
- 按游戏筛选
- 按动态类型筛选
- 按时间筛选
- 按重要级别筛选

---

## 3.3 单游戏详情页 Game Profile
用于回答：**这款游戏现在是什么情况，值得怎么盯？**

页面模块建议：

### A. 基础信息
- 游戏名
- 英文名/别名
- 平台
- 品类
- 研发/发行
- 上线状态
- 上线时间/预计时间
- 核心玩法
- 商业模式
- 目标用户
- 官方入口（官网/TapTap/B站/抖音等）

### B. 核心数据看板
- 热度指数
- 社媒讨论量
- 视频播放/内容量
- 直播热度
- 广告投放趋势
- 排名变化
- 下载/预约/畅销表现（能拿到则展示）

### C. 版本时间线
- 赛季/版本更新时间
- 新职业/新角色/新玩法
- 大型活动
- 回流活动
- 节日节点

### D. 营销动作库
- 达人合作
- 官号内容动作
- 广告素材方向
- 联动活动
- 福利包装
- 周边/品牌事件

### E. 用户口碑与受众
- 用户画像
- 好评点
- 差评点
- 常见吐槽
- 社区敏感点
- 高潜内容方向

### F. 竞品判断
- 直接竞品
- 间接竞品
- 当前优势
- 当前短板
- 值得学习的动作
- 风险点

### G. 运营建议 / 结论
- 一句话判断
- 当前建议重点监控项
- 可借鉴动作
- 我方机会点

---

## 3.4 营销案例页 Marketing Cases
用于回答：**最近有哪些值得复用的案例？**

字段建议：

## 5. V1 信息展示原则

### 5.1 一屏先看结论
每个页面都先给结论，不先堆资料。

### 5.2 数据与判断并存
不是只收集信息，还要保留“所以呢”的判断。

### 5.3 半结构化优先
很多游戏情报不是纯数字，V1 要接受“结构化字段 + 文本结论”混合。

### 5.4 可持续维护
优先搭模板，而不是一次性做炫技页面。

---

## 6. 推荐技术路线（V1）

### 前端
- HTML / CSS / JavaScript 先搭静态原型
- 后续可升级为 React + 图表库

### 数据层
V1 先建议用 JSON 本地数据源：
- games.json
- activities.json
- metrics.json
- cases.json

优点：
- 快速起步
- 容易演示
- 后续可替换成 API / 数据库

### 后续可升级方向
- SQLite / PostgreSQL
- Node.js API
- 自动抓取脚本
- 定时任务同步
- 登录/权限系统

---

## 7. 维护机制建议

### 你（老板）关注
- 哪些游戏进重点池
- 哪些结论需要重点看

### 我负责持续维护
- 模板结构
- 页面字段
- 展示逻辑
- 单游戏档案结构
- 周监控结构
- 内容填充规范

### 后续可半自动化的部分
- 官网/社媒动态收集
- 视频/直播监控
- 版本节点抓取
- 舆情关键词整理

---

## 8. 建议的 V1 重点游戏池
建议先不要铺太大，先做 8-20 款重点游戏：

- 自家产品
- 直接竞品
- 高威胁潜在竞品
- 高营销参考价值产品

先把一小批做深，比一上来铺 100 款更有用。

---

## 9. 我建议的落地顺序

### 第一步
先搭网页原型，确认页面结构与字段。

### 第二步
先录入 3-5 款重点游戏做样板。

### 第三步
再补动态流与案例库。

### 第四步
明确哪些数据靠人工、哪些数据能自动抓。

### 第五步
再决定是否做后台和数据库。

---
